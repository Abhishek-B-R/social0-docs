## Overview

Social0 integrates with external tools in three ways:

- **[CLI](/docs/integrations/cli)** — **preferred** for terminal, CI, and any agent with shell access (`social0`)
- **[MCP](/docs/integrations/mcp)** — only when you **can’t** use the CLI (chat hosts without a shell)
- **[REST API](/docs/api)** — custom backends and scripts that call HTTP directly

## Choose your path

| I want to… | Start here |
|------------|------------|
| Use the terminal, CI, or an agent with shell | [CLI quickstart](/docs/integrations/cli/quickstart) (**preferred**) |
| Use a chat host with **no** shell (Claude.ai / ChatGPT) | [MCP quickstart](/docs/integrations/mcp/quickstart) |
| Build a custom integration or script | [REST API](/docs/api/quickstart) |
| Create or revoke an API key | [Developer settings](/docs/dashboard/api-keys) |
| Connect social accounts first | [Connections](/docs/dashboard/connections) |

## CLI (terminal)

Install with `npm install -g social0`, then `social0 login` with a `sk_live_…` key.

```bash
social0 accounts
social0 publish -c "Hello from the CLI" -p twitter linkedin
```

[Get started with the CLI →](/docs/integrations/cli/quickstart)

## MCP (only if you can’t use the CLI)

Skip MCP if the agent has a shell — use the [CLI](/docs/integrations/cli) instead.

**Remote URL:** add `https://mcp.social0.app/mcp` in chat hosts that only support MCP connectors, then authorize with Social0 OAuth.

**Local stdio:** `npx -y @social0/mcp` with a `sk_live_` key only when the host has neither shell nor remote MCP.

[MCP quickstart →](/docs/integrations/mcp/quickstart)

## REST API

Full programmatic access at `https://api.social0.app/v1`. Authenticate with `Authorization: Bearer sk_live_…`.

- [API Overview](/docs/api) — complete reference, guides, and OpenAPI spec
- [Quickstart](/docs/api/quickstart) — publish your first post in 5 minutes
- [Interactive docs](https://api.social0.app/docs) — live Swagger UI

## Prerequisites

1. Social0 account with at least one [connected platform](/docs/dashboard/connections)
2. For remote MCP: nothing else — OAuth mints a connector key
3. For CLI, local MCP, or REST: an [API key](/docs/dashboard/api-keys) (`sk_live_…`)
