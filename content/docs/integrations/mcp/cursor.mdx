---
title: "Cursor setup"
description: Connect Social0 MCP to Cursor with a remote URL or local npx.
---

## Prefer remote first

If your Cursor build supports HTTP MCP servers, add:

```json
{
  "mcpServers": {
    "social0": {
      "url": "https://mcp.social0.app/mcp"
    }
  }
}
```

Authorize when prompted. No Node.js. No API key in config.

## Local npx (fallback)

Project config (`.cursor/mcp.json`) or **Cursor Settings → MCP**:

```json
{
  "mcpServers": {
    "social0": {
      "command": "npx",
      "args": ["-y", "social0-mcp"],
      "env": {
        "SOCIAL0_API_KEY": "sk_live_your_key_here"
      }
    }
  }
}
```

Requires [Node.js 20+](https://nodejs.org/) and an API key from [Developer settings](/docs/dashboard/api-keys). Reload MCP / restart Cursor, then ask: “Show my connected Social0 accounts.”

## Windsurf

Windsurf uses the same stdio config as Cursor local npx above.
