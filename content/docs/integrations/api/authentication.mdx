## API keys

Create keys in [Developer settings](/docs/dashboard/api-keys). Format: `sk_live_…` (legacy `s0_live_…` still accepted).

```http
Authorization: Bearer sk_live_your_key_here
Content-Type: application/json
```

Keys are shown **once** at creation. Store securely — never commit to git.

## Use with MCP

Use MCP **only if you can’t run the [`social0` CLI](/docs/integrations/cli)**. Prefer remote MCP (`https://mcp.social0.app/mcp`) with OAuth on connector-only hosts. For local stdio:

```json
{
  "mcpServers": {
    "social0": {
      "command": "npx",
      "args": ["-y", "@social0/mcp"],
      "env": {
        "SOCIAL0_API_KEY": "sk_live_your_key_here"
      }
    }
  }
}
```

Unscoped `npx -y social0-mcp` still works as a deprecated alias. If a shell is available, use the [CLI](/docs/integrations/cli) instead of MCP.

See [MCP quickstart](/docs/integrations/mcp/quickstart).

## Security

- One key per integration (Claude vs Cursor vs CI)
- Rotate immediately if leaked
- MCP runs locally for stdio setups — key stays in the host `env` block (or use hosted OAuth with no key in config)
- Revoked keys return `401 invalid_api_key`
