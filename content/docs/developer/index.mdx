## Overview

Social0 has three developer surfaces:

- **CLI** — **preferred** (`social0` on npm) for terminal, CI, and agents with shell access
- **MCP** — only when you **can’t** use the CLI (chat hosts without a shell)
- **REST API** — `https://api.social0.app/v1` for custom backends

## CLI (preferred)

```bash
npm install -g social0
social0 login
social0 accounts
social0 publish -c "Hello from the CLI" -p twitter linkedin
```

## MCP (only if you can’t use the CLI)

Skip MCP when a shell is available — use the CLI above.

**Remote URL:** add `https://mcp.social0.app/mcp` for connector-only hosts, then authorize with Social0 OAuth.

**Local stdio:** `npx -y @social0/mcp` with `SOCIAL0_API_KEY=sk_live_…` only when neither CLI nor remote MCP works.

## Get started

1. Connect at least one account in [Dashboard → Connections](/docs/dashboard/connections)
2. Create a key in [Developer settings](/docs/dashboard/api-keys) for CLI / REST (or use remote MCP OAuth only if you can’t run the CLI)
3. Start with the [CLI quickstart](/docs/integrations/cli/quickstart). Use [MCP](/docs/integrations/mcp/quickstart) only without a shell; [API quickstart](/docs/api/quickstart) for custom backends.

## Tips

- Always prefer the [`social0` CLI](/docs/integrations/cli) when a shell is available. Use MCP only if you can’t run the CLI; use REST for custom backends.
- Don't share your Social0 password with third-party tools — use OAuth or API keys and revoke them anytime.
- Use [webhooks](/docs/api/webhooks) instead of polling for publish status when using the REST API.
