## Overview

Social0 has two developer surfaces:

- **MCP** — let any AI that supports remote MCP connectors post for you (Claude.ai, ChatGPT, Cursor, and others)
- **REST API** — `https://api.social0.app/v1` for backends, CI, and automation

## MCP (AI assistants)

**Remote URL (recommended):** add `https://mcp.social0.app/mcp` in any AI that supports remote MCP connectors, then authorize with Social0 OAuth. No Node.js. No API key in config.

**Local stdio (fallback):** `npx -y social0-mcp` with `SOCIAL0_API_KEY=sk_live_…` (Cursor / Claude Desktop / VS Code when remote isn’t available).

## Get started

1. Connect at least one account in [Dashboard → Connections](/docs/dashboard/connections)
2. Prefer remote MCP: add `https://mcp.social0.app/mcp` as a connector and approve OAuth  
   For local MCP or the REST API: create a key in [Developer settings](/docs/dashboard/api-keys)
3. Follow the [MCP quickstart](/docs/integrations/mcp/quickstart) or [API quickstart](/docs/api/quickstart)

## Tips

- Prefer the remote MCP URL wherever your AI supports it — no Node.js install.
- Don't share your Social0 password with third-party tools — use OAuth or API keys and revoke them anytime.
- Use [webhooks](/docs/api/webhooks) instead of polling for publish status when using the REST API.
